Software Requirement Specification (SRS)
Fleet & Rider Management System (FRMS)
1. Introduction
1.1 Purpose

The purpose of this document is to define the requirements for the Fleet & Rider Management System (FRMS) designed for managing ride-hailing fleet operations in Nepal.

The system will enable fleet owners and operations managers to:

Manage vehicles and riders

Assign riders to vehicles

Track daily ride activity and earnings

Monitor rider productivity

Monitor vehicle performance

Track financial reconciliation

Generate reports and dashboards

Operate using the Nepali calendar (Bikram Sambat)

This document provides a complete description of system behavior, functional requirements, non-functional requirements, and system modules.

1.2 Scope

The Fleet & Rider Management System will support the following:

Vehicle Management

Maintain list of all vehicles

Track vehicle usage and earnings

Track maintenance and operational costs

Rider Management

Maintain list of all riders

Track rider performance

Track rider attendance

Fleet Operations

Assign riders to vehicles

Track daily ride activities

Monitor operational KPIs

Financial Management

Track earnings

Track cash collection

Track incentives and allowances

Analytics

Fleet performance dashboard

Vehicle performance reports

Rider performance reports

Calendar Support

Support Nepali Date (BS)

Support English Date (AD)

Allow filtering using both calendars

1.3 Definitions
Term	Description
Fleet	Group of vehicles owned or managed by the company
Rider	Driver operating a vehicle
Assignment	Linking a rider to a vehicle
Daily Log	Daily operational record of rider activity
Acceptance Rate	Ratio of rides accepted to rides received
Bonus Target	Target number of rides required to earn bonus
Cash Check	Difference between expected and received cash
2. Overall System Description
2.1 System Overview

The system will function as a web-based fleet management platform accessible through desktop and mobile devices.

The platform will consist of:

Fleet dashboard

Vehicle management module

Rider management module

Assignment management module

Daily operations module

Financial tracking module

Maintenance module

Reporting and analytics module

The system will store data centrally and provide real-time insights.

2.2 User Classes
System Owner

Full access to all modules including dashboards, financials, and reports.

Operations Manager

Can manage riders, vehicles, assignments, and daily logs.

Finance/Admin Staff

Can manage cash reconciliation, expenses, salary calculations, and financial reports.

Supervisor

Can monitor daily rider performance and update operational records.

Rider (Optional Portal)

Can view performance, targets, and assigned vehicle.

2.3 Operating Environment

The system will run on:

Web browser (Chrome, Edge, Safari)

Mobile browsers (Android/iOS)

Technology suggestions (for development):

Frontend

Backend

PostgreSQL / MySQL

Nepali date conversion support
